데이터 베이스의 이해

1. 데이터베이스 : 여러 사람들이 공유하고 사용할 목적으로 통합하여 저장한 운영 데이터의 집합

 - 공유 데이터 : 여러 사용자가 필요한 정보를 공동으로 이용할 수 있음
 - 통합 데이터 : 데이터의 중복을 최소화하여 최적화시킴
 - 저장 데이터 : 여러 사용자가 응용 프로그램을 통해 이용할 수 있도록 컴퓨터가 접근 가능한 매체에 저장함
 - 운영 데이터 : 특정 조직을 운영하는 데 필요하고 지속적으로 유지 해야함

2. 데이터베이스 관리 시스템

 - 동시에 실행되는 여러 작업이 올바르게 이루어지도록 해야 함(동시성 제어)
 - 사용 권한이 있는 사용자와 응용 프로그램에게만 데이터 접근을 허용해야 함(보안)
 - 사용 권한이 있는 사용자나 응용 프로그램에 의해 데이터의 검색.삽입.수정.삭제가 이루어진 후에도 데이터가 정확하고 일관되게 유지되어야 함
 - 장애가 발생해도 회복이 가능할 수 있도록 해야 함(백업,롤백)

장점 : 데이터 중복 최소화, 데이터 공유, 보안 항샹, 무결성 유지

단점 : 구축이 복잡함, 라이선스 비용 지불, 백업이 필요

3. 데이터베이스 종류

관계형 데이터베이스
 - 테이블 형식으로 구성
 - sql 언어 사용으로 데이터 조작 가능

NoSQL 비관계형 데이터베이스
 - 많은 데이터 처리를 위한 빅 데이터 분야
 - 이베이, 페이스북

4. 데이터 모델링의 개요

요구사항, 업무는 무엇인가

 - 약속된 표기법을 이용하여 전체 데이터베이스를 표현
 - 데이터들 간의 관계를 바탕으로 업무 시스템을 설계하고 구축함
 - 서비스의 품질과 응용프로그램에 영향을 줌

모델링의 단계
 - 개념적 데이터 모델링 -> ex 아파트 설계도 작성같은
 - 논리적 데이터 모델링 -> 모델하우스
 - 물리적 데이터 모델링 -> 아파트 설계

 개념적 데이터 모델링
  구성요소 -> 개체, 속성, 관계

 	개체: 정보를 저장하기 위한 모든것 -> 학생, 학과, 과목 등
	  개체에는 고유한 이름과 하나 이상의 속성을 가짐

	개체 타입 : 개체의 이름과 속성을 정의한 것
	개체 인스턴스 : 개체를 구성하는 속성이 실체 값을 가지는 개체
	개체 집합 : 개체 인스턴스의 집합, 실제 데이터베이스에 저장됨

	 E-R 다이어그램에서는 사각형

	속성: 개체가 가지는 고유한 특성, 의미상 더 이상 분리되지 않는 최소의 데이터 단위
	 각각의 속성은 현실 세계의 존재하는 값을 가짐

	 단일 값 속성
	 E-R 다이어그램에서는 타원형
	 다중 값 속성
	 E-R 다이어그램에서는 이중타원형
	 복합 속성

	 키 속성
	 E-R 다이어그램에서는 밑줄을 그어서 표시

	관계: 개체와 개체끼리 의미있는 연관성, 개체간의 수학적 관계
	 E-R 다이어그램에서는 마름모

	카디널리티
	 1:1 관계
	 E-R 다이어그램에서는 실선 위에 1을 표기
	 1:N 관계
	 E-R 다이어그램에서는 실선 위에 1,N을 표기
	 M:N 관계
	 E-R 다이어그램에서는 실선 위에 N,M을 표기

 논리적 데이터 모델링
   개념적 설계 과정을 통해 도출된 개념적 구조를 기반으로 데이터베이스 관리 시스템에 적합한 논리적 구조로 표현하는 과정

	1 모든 개체는 릴레이션으로 변환 
	2 M : N 관계는 릴레이션으로 변환 
	3 다중 값 속성은 릴레이션으로 변환
	4 1 : N 관계, 1 : 1 관계는 외래키로 표

 물리적 데이터 모델링
   선정한 데이터베이스 관리 시스템에 적합하도록 논리적 구조를 기반으로 물리적 구조를 설계하는 과정

   릴레이션 : 테이블(Table) , 속성 : 열(Column) , 투플 : 행(Row)
   각 테이블과 열의 이름, 데이터타입과 크기, 제약조건 등을 지정
